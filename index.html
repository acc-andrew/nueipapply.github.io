NUEiP application GoLang 工程師培訓營

 大同大學資訊工程研究所畢業。6~7 年的程式開發經驗，開發過汽車衛星導航系統、手機備份軟體。

 性格嚴謹，學習新事物喜歡追根究底。優點是容易學習特色，但缺點是產出效率較差。
 為了提高效率，我找到測試驅動開發(TDD)來提高軟體品質及產生效率。

 軟體工程師是與時俱進的工作，自學能力是必備技能。而學習的目標則依照趨勢而定。
自學了 ASP.NET C# WebForm, MVC, WebAPI, Bootstrap, jQuery
網路爬蟲、機器學習、深度學習、人臉識別。

 GoLang 被譽為 21 世紀的 C 語言，我也在 YouTube 上學習 Go+Gin RESTful 結構的會員登入功能。
希望可藉此訓練營學習到業界的網路程式設計。

第三題：為什麼想成為工程師（請以自我期許帶入：前端、後端、或全端工程師）

 進入大學之前，我就開始在業界當軟體工程師。開發過汽車衛星導航系統。
當時沒有專人栽培，我只能從原文雜誌、電腦書局自行找尋程式設計、C++/MFC 開發、Win32 系統架構等技術內容。
 進入了資訊工程系與資訊工程研究所，使我對演算法有深刻的理解，也可以應用在軟體開發上。
工程師就是善用知識，親手實作的職業。同時兼顧理性與感性。
我一路由 C, C++, C#, SQL 資料庫學習程式設計，本以為成為後端工程師即可。
後來接觸了台灣業界，才知道學後端的工程師也需要學前端的工作。現在以成為全端工程師為自我期許。

 也參加經濟部工業局舉辦的 深度學習、聊天機器人、推薦系統等課程。

104 相關職缺推薦系統等專題實作。
 網路學習課程 Udemy 也持續學習。

MS-SQL 資料庫，並在程式開發中加入測試驅動開發(Test-Driven Development:TDD)。

第五題：對於工程師的職業想像是什麼？

 對於成為軟體工程師已經超過 10 年的我，原本我對工程師的職業的觀念就是一個理性導向
有限時間內完成最高品質、最少錯誤的工程作品。
但在近期的 使用者互動、動態網頁(Bootstrap, jQuery, AJAX)、彈性配置系統環境的 Docker, Kubernetes, WebAPI
網路安全(SQL injection, )
已經漸漸看出無論是在 主從式網站架構、API 式彈性架構、作業系統可攜式等各種應用層面，
軟體工程師都需要具備 彈性考慮、使用者觀點、商業資源規劃 等多種專業的結合。


第六題：如果參與這個計畫，會怎麼安排學習時間？

  一週 40~60 小時，相當於一天 8~12 小時。其實時間已經很長。
我會提高學習效率來面對長時間的學習計畫。
一天之中，清晨是學習效率最好的時候。我會將 早晨分成 3~4 段學習時間。
早晨先學習議題結論。下午及晚上學習效率較低，就以操作流程作為學習內容。


第七題：如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？

 困難的地方應該在：
1. Mutual Exclusion、Goroutine、Process 等多執行續(multi-threading) 等同步控制問題
在自己的電腦上或 Google Cloud Platform 安裝Go 語言開發環境，直接實作學習；
也可以到 StackOverflow 等網路上找工程師們寫過的經驗；還可以利用 C# 撰寫等同的程式來測試與驗證。

2. MySQL、資料庫正規化
工程師的經驗是以實作學來的，可以在自己的電腦上安裝 MySQL，從網路或 Udemy 找相關課程實際來實作。
也可以到 StackOverflow 等網路上找工程師們寫過的程式來練習。

3. GCP 雲端服務：compute engine、Docker 應用
可以上 Google Cloud Platform 設定 compute engine 環境，實作學習經驗
Docker 可以在自己的電腦上安裝 Docker 環境，實作學習經驗。
